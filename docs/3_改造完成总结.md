def _dict_to_relation(self, rel_dict: Dict) -> Relation:
    """将 dict 格式的关系转换为 Relation 对象

    注意：dict 格式遵循 rel JSON 约定（from=主键表, to=外键表），
    但 Relation 对象约定是 source=外键表, target=主键表，
    因此需要交换 from/to
    """
    # 创建 Relation 对象（交换 from/to 以符合 Relation 约定）
    return Relation(
        relationship_id=rel_dict["relationship_id"],
        source_schema=rel_dict["to_table"]["schema"],  # 外键表
        source_table=rel_dict["to_table"]["table"],
        source_columns=to_columns,
        target_schema=rel_dict["from_table"]["schema"],  # 主键表
        target_table=rel_dict["from_table"]["table"],
        target_columns=from_columns,
        relationship_type="inferred" if rel_dict.get("discovery_method") == "llm_assisted" else "foreign_key",
        cardinality=self._flip_cardinality(rel_dict.get("cardinality", "N:1")),
        composite_score=rel_dict.get("composite_score"),
        score_details=rel_dict.get("metrics"),
        inference_method=rel_dict.get("discovery_method")
    )

## 关键设计决策

### 1. 语义转换处理

**问题：** dict 格式和 Relation 对象的语义相反
- dict: `from=主键表, to=外键表`
- Relation: `source=外键表, target=主键表`

**解决方案：**
- `_dict_to_relation()`: dict → Relation 时交换 from/to
- `_relation_to_dict()`: Relation → dict 时交换 source/target
- 同时翻转 cardinality（N:1 ↔ 1:N）

### 2. 字段名映射

| dict 格式 | Relation 对象 |
|----------|--------------|
| `metrics` | `score_details` |
| `discovery_method` | `inference_method` |
| `from_table` | `target_*` (主键表) |
| `to_table` | `source_*` (外键表) |

### 3. 向后兼容

- RelationshipWriter 的 `generated_by` 默认值为 `"rel"`，确保 `--step rel` 不受影响
- `extra_statistics` 为可选参数，默认为 `None`
- CQL 生成器只读取 relationships 数组中的必需字段，新字段不影响

## 关键 Bug 修复

### Bug 1: `_filter_existing_fks()` 单向检查失效 🔴

**严重性：** 高优先级
**影响：** 物理外键可能进入 LLM 评分阶段，造成资源浪费和潜在的重复关系

**问题描述：**
代码错误地假设 `relationship_id` 是双向一致的，但实际上它是有方向的（source->target 格式）。当 LLM 返回与物理外键相反的方向时，过滤失效。

**修复：** `metaweave/core/relationships/llm_relationship_discovery.py:898-960`
```python
# 计算正向和反向两个 relationship_id
forward_rel_id = MetadataRepository.compute_relationship_id(
    source=from_table, target=to_table, ...
)
reverse_rel_id = MetadataRepository.compute_relationship_id(
    source=to_table, target=from_table, ...  # 交换方向
)

# 检查任一方向匹配
if forward_rel_id in fk_relationship_ids or reverse_rel_id in fk_relationship_ids:
    # 跳过这个候选（它是物理外键）
```

### Bug 2: `_dict_to_relation()` 使用错误的 relationship_id 方向 🔴

**严重性：** 高优先级
**影响：** 阶段7去重完全失效，所有物理外键会被重复输出（一次作为 foreign_key，一次作为 llm_assisted）

**问题描述：**
方法直接复制 dict 的 `relationship_id`（使用 PK->FK 方向），但 Relation 对象应使用 FK->PK 方向。导致物理外键和 LLM 推断关系的 ID 永远不匹配。

**修复：** `metaweave/core/relationships/llm_relationship_discovery.py:616-663`
```python
def _dict_to_relation(self, rel_dict: Dict) -> Relation:
    # 提取列名
    if rel_dict["type"] == "single_column":
        from_columns = [rel_dict["from_column"]]
        to_columns = [rel_dict["to_column"]]
    else:
        from_columns = rel_dict["from_columns"]
        to_columns = rel_dict["to_columns"]

    # ✅ 重新计算 relationship_id，使用 Relation 方向（FK->PK）
    relationship_id = MetadataRepository.compute_relationship_id(
        source_schema=rel_dict["to_table"]["schema"],      # FK 表
        source_table=rel_dict["to_table"]["table"],
        source_columns=to_columns,
        target_schema=rel_dict["from_table"]["schema"],    # PK 表
        target_table=rel_dict["from_table"]["table"],
        target_columns=from_columns,
        rel_id_salt=self.repo.rel_id_salt
    )

    return Relation(
        relationship_id=relationship_id,  # ✅ 使用重新计算的 ID
        # ...
    )
```
