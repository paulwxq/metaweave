# MetaWeave 迁移后运行指南（WSL + uv）

本指南适用于已从 `nl2sql_v3/` **复制迁移**到 `/mnt/c/Projects/cursor_2025h2/metaweave/` 的独立项目版本。

## 1. 环境准备

### 1.1 Python / uv

- Python：3.12
- 包管理与虚拟环境：`uv`

WSL 推荐使用虚拟环境目录：`.venv-wsl`

```bash
cd /mnt/c/Projects/cursor_2025h2/metaweave
uv venv .venv-wsl --python 3.12
source .venv-wsl/bin/activate
uv sync --active
```

说明：
- `uv sync --active` 会把依赖安装到“当前激活的虚拟环境”中（即 `.venv-wsl`）。
- 如果不加 `--active`，`uv` 可能会改用默认项目环境目录（通常是 `.venv`），从而装到你不希望的位置。

## 2. 配置文件与需要修改的内容

### 2.1 `.env`（必需）

项目根目录提供了模板：`/mnt/c/Projects/cursor_2025h2/metaweave/.env.example`  
你需要创建：`/mnt/c/Projects/cursor_2025h2/metaweave/.env`

`.env` 通常需要包含（按你实际环境填写）：

- PostgreSQL：`DB_HOST` `DB_PORT` `DB_NAME` `DB_USER` `DB_PASSWORD`
- Neo4j：`NEO4J_URI` `NEO4J_USER` `NEO4J_PASSWORD` `NEO4J_DATABASE`
- Milvus：`MILVUS_HOST` `MILVUS_PORT` `MILVUS_DATABASE`（可选：`MILVUS_USER` `MILVUS_PASSWORD`）
- LLM：
  - Qwen：`DASHSCOPE_API_KEY` `DASHSCOPE_BASE_URI`
  - DeepSeek：`DEEPSEEK_API_KEY` `DEEPSEEK_BASE_URI`

注意：
- 不要提交 `.env`；已在 `.gitignore` 中忽略。
- `services/config_loader.py` 会自动加载项目根目录的 `.env`。

### 2.2 `configs/config.yaml`（全局配置，Loader 会用到）

路径：`/mnt/c/Projects/cursor_2025h2/metaweave/configs/config.yaml`

用途：
- 作为 **全局配置**，供 `services/db/*`、`services/vector_db/*` 读取 DB/Neo4j/Milvus 连接信息
- 其中的 `${VAR_NAME}` 会从 `.env` 读取并替换

你通常需要修改：
- `database.*`（若你的 DB 信息与默认不同）
- `neo4j.*`
- `vector_database.*`（Milvus 或 PgVector 的连接信息）

### 2.3 `configs/metadata_config.yaml`（元数据生成主配置）

路径：`/mnt/c/Projects/cursor_2025h2/metaweave/configs/metadata_config.yaml`

用途：
- 控制元数据生成流程：采样、LLM、embedding、输出目录等

你通常需要修改/关注：
- `database.schemas`：要抽取的 schema 列表
- `database.exclude_tables`：排除表规则
- `llm.active` / `llm.providers.*`：选择 qwen/deepseek 与模型参数
- `embedding.*`：embedding provider 与模型维度等
- `vector_database.*`：MetaWeave 自己使用的向量库配置（同样依赖 `.env`）
- `output.*`：
  - `output_dir: output`
  - `json_directory: output/json`
  - `md_directory: output/md`（若存在）
  - `rel_directory: output/rel`
  - `cql_directory: output/cql`
  - `cache_file: cache/comment_cache.json`
  - `log_file: logs/metadata.log`

### 2.4 `configs/loader_config.yaml`（加载器配置）

路径：`/mnt/c/Projects/cursor_2025h2/metaweave/configs/loader_config.yaml`

用途：
- `metaweave load --type ...` 时使用

重点：
- `cql_loader.input_file`：默认 `output/cql/import_all.cypher`
- `cql_loader.neo4j.use_global_config: true` 时，会使用 `configs/config.yaml` 的 `neo4j` 段
- 顶层 `metadata_config_file`：建议显式配置（默认回退 `configs/metadata_config.yaml`）
- `table_schema_loader.*`：默认从 `output/md` 与 `output/json` 读取，且 `collection_name` 必填
- `dim_loader.*`：`config_file` 指向维表配置，且 `collection_name` 必填

### 2.5 `configs/db_domains.yaml`（可选：domain 功能）

路径：`/mnt/c/Projects/cursor_2025h2/metaweave/configs/db_domains.yaml`

用途：
- domain/cross-domain 功能需要它（包含 `database.description` 与 `domains` 列表）

常见流程：
1) 先填写 `database.description`
2) 执行 `--generate-domains` 生成 `domains` 列表

### 2.6 `configs/dim_tables.yaml`（可选：维表配置）

路径：`/mnt/c/Projects/cursor_2025h2/metaweave/configs/dim_tables.yaml`

用途：
- `dim_value` loader 使用，用于把维表值写入向量数据库

注意：
- 可先自动生成框架，但必须手工补充 `embedding_col`（要向量化的列）。

## 3. 运行命令（迁移后版本）

下面命令默认在项目根目录执行，且使用 `.venv-wsl`：

```bash
cd /mnt/c/Projects/cursor_2025h2/metaweave
source .venv-wsl/bin/activate
```

### 3.1 查看 CLI

```bash
metaweave --help
metaweave metadata --help
metaweave load --help
metaweave dim_config --help
```

### 3.2 元数据生成（metadata）

完整流程（常用）：

```bash
metaweave metadata --config configs/metadata_config.yaml --step all
```

按步骤执行（调试/分阶段）：

```bash
metaweave metadata --config configs/metadata_config.yaml --step ddl
metaweave metadata --config configs/metadata_config.yaml --step json
metaweave metadata --config configs/metadata_config.yaml --step rel
metaweave metadata --config configs/metadata_config.yaml --step cql
metaweave metadata --config configs/metadata_config.yaml --step md
```

注意：
- `--step cql` 会读取 `output/json` 与 `output/rel` 作为输入；如果你只执行 `--step cql` 而没有先生成 `json/rel`，会报 `JSON 目录不存在` 或找不到关系文件的错误。
  - 推荐顺序：先 `--step json`，再 `--step rel`，最后 `--step cql`（或直接 `--step all`）。

LLM 增强步骤（需要 LLM Key）：

```bash
metaweave metadata --config configs/metadata_config.yaml --step json_llm
metaweave metadata --config configs/metadata_config.yaml --step rel_llm
metaweave metadata --config configs/metadata_config.yaml --step cql_llm
```

只处理指定 schema / 表：

```bash
metaweave metadata --config configs/metadata_config.yaml --schemas public
metaweave metadata --config configs/metadata_config.yaml --tables table_a,table_b
```

### 3.3 domain 相关（可选）

生成 domains（会写回 `configs/db_domains.yaml`，生成完成后退出）：

```bash
metaweave metadata --config configs/metadata_config.yaml --generate-domains
```

按 domain 过滤生成（示例）：

```bash
metaweave metadata --config configs/metadata_config.yaml --domain all
metaweave metadata --config configs/metadata_config.yaml --domain A,B
metaweave metadata --config configs/metadata_config.yaml --cross-domain
```

### 3.4 生成维表配置（dim_config）

```bash
metaweave dim_config --generate --config configs/metadata_config.yaml --output configs/dim_tables.yaml
```

生成后请手工编辑 `configs/dim_tables.yaml`，补充每张维表的 `embedding_col`。

### 3.5 加载到目标数据库（load）

加载 Neo4j CQL（`output/cql/import_all.cypher` → Neo4j）：

```bash
metaweave load --type cql --config configs/loader_config.yaml
```

可选：加载前清空数据库（谨慎）：

```bash
metaweave load --type cql --config configs/loader_config.yaml --clean
```

加载 table_schema 到向量库（需要 `output/md` 与 `output/json` 已存在）：

```bash
metaweave load --type table_schema --config configs/loader_config.yaml
```

加载维表值（需要 `configs/dim_tables.yaml` 已配置，且向量库可用）：

```bash
metaweave load --type dim_value --config configs/loader_config.yaml
```
