Holistic Primary Key and Foreign Key Detection - 主外键关系自动发现的工业级实现

📚 算法概览

HoPF (Holistic Primary Key and Foreign Key Detection) 是一种自动化发现数据库表之间主外键关系的先进算法。它通过三个阶段的处理流程，从原始数据库Schema中推断出潜在的主键和外键关系，准确率可达85-95%。

🎯 核心特性

  • 无需人工标注
  • 支持复合主键
  • 多维度评分机制
  • 自动冲突解决

⚡ 性能指标

准确率 85-95%
100张表处理时间 ~5分钟

🔧 适用场景

  • 数据仓库逆向工程
  • 遗留系统文档重建
  • 数据血缘分析
  • 数据质量评估
1 阶段一: 候选生成 (Candidate Generation)

🔍 目标

发现数据库中所有可能的唯一列组合(UCC)和包含依赖关系(IND)，为后续评分提供候选集。

📊 1.1 唯一列组合(UCC)发现

什么是UCC?

Unique Column Combination - 能够唯一标识表中每一行的列或列组合。

数学定义: 对于表T的列集C, 如果 |π_C(T)| = |T|, 则C是UCC

算法步骤

Step 1: 单列唯一性检测
-- 检测order_id是否唯一
SELECT 
    COUNT(*) as total_rows,
    COUNT(DISTINCT order_id) as distinct_values,
    COUNT(DISTINCT order_id) * 1.0 / COUNT(*) as uniqueness_ratio
FROM orders;

-- 结果: 
-- total_rows=1000, distinct_values=1000, uniqueness_ratio=1.0
-- 结论: order_id 是 UCC ✓
Step 2: 组合列唯一性检测
-- 检测(customer_id, order_date)组合是否唯一
SELECT 
    customer_id, 
    order_date, 
    COUNT(*) as cnt
FROM orders
GROUP BY customer_id, order_date
HAVING COUNT(*) > 1;

-- 结果: 无重复
-- 结论: (customer_id, order_date) 是 UCC ✓
Step 3: 剪枝优化

剪枝规则:

  • 如果 {A} 是UCC, 则 {A, B} 必然是UCC (无需检测)
  • 如果 {A, B} 不是UCC, 则 {A} 和 {B} 都不是UCC
  • 优先检测列数少的组合
时间复杂度: O(2^n) → O(n^2) (剪枝后)

示例输出

表名 UCC候选 列数 唯一性
orders {order_id} 1 1.000
customers {customer_id} 1 1.000
order_items {item_id} 1 1.000
order_items {order_id, product_id} 2 1.000 
def find_uccs(table_name, connection, max_columns=3):
    """
    发现表的所有唯一列组合
    
    Args:
        table_name: 表名
        connection: 数据库连接
        max_columns: 最大组合列数
    
    Returns:
        List[Tuple]: UCC列表, 每个元素为(列组合, 唯一性比例)
    """
    uccs = []
    
    # 获取表的所有列
    columns = get_table_columns(table_name, connection)
    row_count = get_row_count(table_name, connection)
    
    # Step 1: 检测单列UCC
    for col in columns:
        distinct_count = execute_query(
            f"SELECT COUNT(DISTINCT {col}) FROM {table_name}",
            connection
        )[0][0]
        
        uniqueness = distinct_count / row_count
        
        if uniqueness >= 0.99:  # 唯一性阈值
            uccs.append(([col], uniqueness))
            print(f"✓ Found UCC: {col} (uniqueness={uniqueness:.3f})")
    
    # Step 2: 检测组合UCC (使用剪枝)
    if len(uccs) < len(columns):  # 如果不是所有列都是UCC
        for n in range(2, max_columns + 1):
            candidates = generate_combinations(columns, n, uccs)
            
            for combo in candidates:
                if has_superset_ucc(combo, uccs):
                    # 剪枝: 已有子集是UCC, 跳过
                    continue
                
                cols_str = ', '.join(combo)
                query = f"""
                    SELECT COUNT(*) as total,
                           COUNT(DISTINCT {cols_str}) as distinct_cnt
                    FROM {table_name}
                """
                result = execute_query(query, connection)[0]
                
                if result[0] == result[1]:  # 完全唯一
                    uccs.append((combo, 1.0))
                    print(f"✓ Found composite UCC: {combo}")
    
    return uccs


def generate_combinations(columns, n, existing_uccs):
    """生成候选组合并剪枝"""
    from itertools import combinations
    
    # 排除已经是UCC的列
    ucc_columns = set()
    for ucc, _ in existing_uccs:
        if len(ucc) == 1:
            ucc_columns.add(ucc[0])
    
    remaining_columns = [c for c in columns if c not in ucc_columns]
    
    return list(combinations(remaining_columns, n))

🔗 1.2 包含依赖(IND)发现

什么是IND?

Inclusion Dependency - 表A的某列(组)的所有值都包含在表B的某列(组)中。

数学定义: A[L1] ⊆ B[L2] 表示 ∀t∈A, ∃s∈B 使得 t[L1] = s[L2]

算法步骤

Step 1: 基本包含检测
-- 检测 orders.customer_id ⊆ customers.customer_id
SELECT COUNT(DISTINCT o.customer_id) as fk_distinct,
       COUNT(DISTINCT CASE 
           WHEN EXISTS (
               SELECT 1 FROM customers c 
               WHERE c.customer_id = o.customer_id
           ) THEN o.customer_id 
       END) as matched_distinct
FROM orders o;

-- 如果 fk_distinct = matched_distinct, 则存在包含依赖
Step 2: 布隆过滤器优化

对于大规模数据，使用布隆过滤器快速预检测:

from pybloom_live import BloomFilter

def check_inclusion_with_bloom(table_a, col_a, table_b, col_b, conn):
    """使用布隆过滤器快速检测包含依赖"""
    
    # Step 1: 构建主表的布隆过滤器
    bf = BloomFilter(capacity=1000000, error_rate=0.001)
    
    query_b = f"SELECT DISTINCT {col_b} FROM {table_b}"
    for row in execute_query(query_b, conn):
        bf.add(str(row[0]))
    
    # Step 2: 检测外表值
    query_a = f"SELECT DISTINCT {col_a} FROM {table_a}"
    not_included_count = 0
    total_count = 0
    
    for row in execute_query(query_a, conn):
        total_count += 1
        if str(row[0]) not in bf:
            not_included_count += 1
            if not_included_count > 100:  # 早停
                return False, 0.0
    
    # Step 3: 精确验证 (布隆过滤器可能误报)
    if not_included_count == 0:
        inclusion_ratio = verify_exact_inclusion(
            table_a, col_a, table_b, col_b, conn
        )
        return inclusion_ratio >= 0.95, inclusion_ratio
    
    return False, 0.0
时间复杂度: O(n+m) vs O(n×m) (传统方法)

性能优化策略

📊 采样策略

  • 小表(<10K行): 全量检测
  • 中表(10K-1M): 10%采样预检
  • 大表(>1M): 1%采样 + 统计信息

⚡ 并行化

  • 表对检测独立并行
  • 使用Spark/Dask处理
  • 分区并行聚合

📊 2.1 六维度评分体系

维度1: 唯一性得分 (权重30%)

衡量主键候选列的唯一性程度

def calculate_uniqueness_score(distinct_count, total_count):
    """
    计算唯一性得分
    
    Args:
        distinct_count: 唯一值数量
        total_count: 总行数
    
    Returns:
        float: 0-10分
    """
    uniqueness_ratio = distinct_count / total_count
    
    if uniqueness_ratio >= 0.99:
        return 10.0  # 几乎完美
    elif uniqueness_ratio >= 0.95:
        return 8.0   # 高唯一性
    elif uniqueness_ratio >= 0.90:
        return 5.0   # 中等唯一性
    else:
        return 0.0   # 不适合做主键
示例: customers.customer_id: 500唯一值/500行 = 1.0 → 10分 orders.customer_id: 300唯一值/1000行 = 0.3 → 0分
维度2: 包含度得分 (权重25%)

衡量外键值在主键值域中的覆盖程度

def calculate_inclusion_score(fk_values, pk_values):
    """
    计算包含度得分
    
    Args:
        fk_values: 外键的唯一值集合
        pk_values: 主键的唯一值集合
    
    Returns:
        float: 0-10分
    """
    intersection = len(fk_values & pk_values)
    inclusion_ratio = intersection / len(fk_values)
    
    if inclusion_ratio == 1.0:
        return 10.0  # 完美包含
    elif inclusion_ratio >= 0.95:
        return 8.0   # 少量孤儿值
    elif inclusion_ratio >= 0.80:
        return 5.0   # 需要人工确认
    else:
        return 0.0   # 可能不是外键
维度3: 命名规范得分 (权重20%)

基于命名模式识别主外键关系

import re

def calculate_naming_score(column_name, is_pk_candidate=True):
    """
    计算命名规范得分
    
    Args:
        column_name: 列名
        is_pk_candidate: 是否主键候选
    
    Returns:
        float: 0-10分
    """
    pk_patterns = [
        r'.*_id$',      # user_id, order_id
        r'^id$',        # id
        r'.*_key$',     # order_key
        r'.*_no$',      # order_no
        r'.*_code$',    # product_code
    ]
    
    fk_patterns = [
        r'^fk_.*',      # fk_customer
        r'.*_id$',      # customer_id (在非主表中)
        r'.*_ref$',     # customer_ref
    ]
    
    patterns = pk_patterns if is_pk_candidate else fk_patterns
    
    for pattern in patterns:
        if re.match(pattern, column_name.lower()):
            return 10.0
    
    # 检查表名关联
    # 如 customers表中的customer_id
    table_name = get_table_name_from_context()
    if table_name.rstrip('s') in column_name.lower():
        return 8.0
    
    return 0.0
维度4: 列数得分 (权重15%)

优先选择列数最少的主键组合

def calculate_minimality_score(column_count):
    """最小性得分 - 优先单列主键"""
    scores = {
        1: 10.0,  # 单列主键
        2: 7.0,   # 双列复合主键
        3: 4.0,   # 三列复合主键
        4: 2.0,   # 四列及以上
    }
    return scores.get(column_count, 1.0)
维度5: 基数比例得分 (权重5%)

检查主外键的唯一值数量比例

def calculate_cardinality_score(fk_distinct, pk_distinct):
    """
    基数比例得分
    
    典型的一对多关系: FK唯一值 ≈ PK唯一值
    """
    ratio = fk_distinct / pk_distinct
    
    if 0.8 <= ratio <= 1.2:
        return 10.0  # 正常一对多
    elif 0.5 <= ratio < 0.8 or 1.2 < ratio <= 2.0:
        return 6.0   # 可能的关联
    else:
        return 2.0   # 关系不明确
维度6: 类型匹配度 (权重5%)

检查数据类型兼容性

def calculate_type_score(fk_type, pk_type):
    """数据类型匹配得分"""
    
    # 完全匹配
    if fk_type == pk_type:
        return 10.0
    
    # 兼容类型
    compatible_pairs = [
        ('INT', 'BIGINT'),
        ('VARCHAR(50)', 'VARCHAR(100)'),
        ('DECIMAL(10,2)', 'DECIMAL(18,2)'),
    ]
    
    for type1, type2 in compatible_pairs:
        if (fk_type == type1 and pk_type == type2) or \
           (fk_type == type2 and pk_type == type1):
            return 8.0
    
    # 需要转换
    convertible_pairs = [
        ('INT', 'VARCHAR'),
        ('DATE', 'TIMESTAMP'),
    ]
    
    for type1, type2 in convertible_pairs:
        if (fk_type in type1 and pk_type in type2) or \
           (fk_type in type2 and pk_type in type1):
            return 6.0
    
    return 0.0  # 不兼容

🧮 2.2 综合评分计算

def calculate_final_score(candidate):
    """
    计算候选关系的最终得分
    
    Args:
        candidate: {
            'fk_table': 'orders',
            'fk_column': 'customer_id',
            'pk_table': 'customers',
            'pk_column': 'customer_id',
            'stats': {...}
        }
    
    Returns:
        float: 0-10的最终得分
    """
    weights = {
        'uniqueness': 0.30,
        'inclusion': 0.25,
        'naming': 0.20,
        'minimality': 0.15,
        'cardinality': 0.05,
        'type_match': 0.05,
    }
    
    scores = {
        'uniqueness': calculate_uniqueness_score(
            candidate['stats']['pk_distinct'],
            candidate['stats']['pk_total']
        ),
        'inclusion': calculate_inclusion_score(
            candidate['stats']['fk_values'],
            candidate['stats']['pk_values']
        ),
        'naming': (
            calculate_naming_score(candidate['pk_column'], True) +
            calculate_naming_score(candidate['fk_column'], False)
        ) / 2,
        'minimality': calculate_minimality_score(
            len(candidate['fk_column'].split(','))
        ),
        'cardinality': calculate_cardinality_score(
            candidate['stats']['fk_distinct'],
            candidate['stats']['pk_distinct']
        ),
        'type_match': calculate_type_score(
            candidate['stats']['fk_type'],
            candidate['stats']['pk_type']
        ),
    }
    
    # 加权求和
    final_score = sum(
        scores[dim] * weights[dim] 
        for dim in scores
    )
    
    # 记录评分明细
    candidate['score_breakdown'] = scores
    candidate['final_score'] = final_score
    
    return final_score

🔝 2.3 Top K筛选

def filter_top_candidates(candidates, threshold=6.0, top_k=None):
    """
    筛选高分候选
    
    Args:
        candidates: 所有候选列表
        threshold: 最低分数阈值
        top_k: 保留前K个 (None表示保留所有达标的)
    
    Returns:
        List: 筛选后的候选列表
    """
    # 按得分降序排序
    sorted_candidates = sorted(
        candidates,
        key=lambda x: x['final_score'],
        reverse=True
    )
    
    # 过滤低分
    filtered = [
        c for c in sorted_candidates 
        if c['final_score'] >= threshold
    ]
    
    # Top K
    if top_k:
        filtered = filtered[:top_k]
    
    return filtered

⚠️ 3.1 冲突检测与解决

冲突场景

同一个外键列可能匹配到多个主键表,需要选择最佳匹配。

示例冲突: orders.id → customers.id (得分9.5) orders.id → products.id (得分9.3) 问题: orders.id应该关联到哪张表? 
def resolve_conflicts(candidates):
    """
    解决外键冲突
    
    策略:
    1. 保留得分最高的关系
    2. 其他关系标记为"待确认"
    3. 如果得分接近,检查语义相似度
    """
    # 按外键分组
    fk_groups = {}
    for candidate in candidates:
        fk_key = f"{candidate['fk_table']}.{candidate['fk_column']}"
        if fk_key not in fk_groups:
            fk_groups[fk_key] = []
        fk_groups[fk_key].append(candidate)
    
    resolved = []
    conflicts = []
    
    for fk_key, group in fk_groups.items():
        if len(group) == 1:
            # 无冲突
            resolved.append(group[0])
        else:
            # 有冲突,按得分排序
            sorted_group = sorted(
                group, 
                key=lambda x: x['final_score'], 
                reverse=True
            )
            
            best = sorted_group[0]
            others = sorted_group[1:]
            
            # 检查得分差距
            if len(others) > 0 and \
               (best['final_score'] - others[0]['final_score']) < 0.5:
                # 得分接近,使用语义相似度辅助判断
                best = select_by_semantic_similarity(sorted_group)
            
            best['status'] = 'accepted'
            resolved.append(best)
            
            # 其他候选标记为待确认
            for candidate in others:
                candidate['status'] = 'pending_review'
                candidate['conflict_reason'] = \
                    f"与 {best['pk_table']}.{best['pk_column']} 冲突"
                conflicts.append(candidate)
    
    return resolved, conflicts


def select_by_semantic_similarity(candidates):
    """基于表名和列名的语义相似度选择"""
    from difflib import SequenceMatcher
    
    def similarity(a, b):
        return SequenceMatcher(None, a, b).ratio()
    
    best_candidate = None
    best_sim = 0
    
    for candidate in candidates:
        # 计算表名相似度
        fk_table = candidate['fk_table'].lower()
        pk_table = candidate['pk_table'].lower()
        
        # orders vs customers, orders vs products
        sim = similarity(fk_table, pk_table)
        
        if sim > best_sim:
            best_sim = sim
            best_candidate = candidate
    
    return best_candidate

🔄 3.2 循环引用检测

循环引用问题

A→B→C→A 形成环,在数据库中通常是设计错误。

示例环路: table_A.ref_b → table_B.id table_B.ref_c → table_C.id table_C.ref_a → table_A.id 这会导致无法确定插入顺序! 
def detect_cycles(relationships):
    """
    检测外键关系中的循环引用
    
    使用DFS算法检测有向图中的环
    """
    # 构建关系图
    graph = {}
    for rel in relationships:
        fk_table = rel['fk_table']
        pk_table = rel['pk_table']
        
        if fk_table not in graph:
            graph[fk_table] = []
        graph[fk_table].append((pk_table, rel))
    
    visited = set()
    rec_stack = set()
    cycles = []
    
    def dfs(node, path):
        visited.add(node)
        rec_stack.add(node)
        path.append(node)
        
        if node in graph:
            for neighbor, rel in graph[node]:
                if neighbor not in visited:
                    if dfs(neighbor, path.copy()):
                        return True
                elif neighbor in rec_stack:
                    # 发现环!
                    cycle_start = path.index(neighbor)
                    cycle = path[cycle_start:] + [neighbor]
                    cycles.append({
                        'cycle': cycle,
                        'relationships': [
                            r for r in relationships
                            if r['fk_table'] in cycle and 
                               r['pk_table'] in cycle
                        ]
                    })
                    return True
        
        rec_stack.remove(node)
        return False
    
    # 检测所有节点
    for node in graph:
        if node not in visited:
            dfs(node, [])
    
    return cycles


def break_cycles(relationships, cycles):
    """
    打破循环引用
    
    策略: 移除环中得分最低的关系
    """
    relationships_to_remove = []
    
    for cycle_info in cycles:
        rels_in_cycle = cycle_info['relationships']
        
        # 找到得分最低的关系
        min_score_rel = min(
            rels_in_cycle, 
            key=lambda x: x['final_score']
        )
        
        # 标记移除
        min_score_rel['status'] = 'removed_by_cycle_detection'
        min_score_rel['cycle_warning'] = \
            f"在环路 {' → '.join(cycle_info['cycle'])} 中"
        
        relationships_to_remove.append(min_score_rel)
    
    # 从关系列表中移除
    cleaned_relationships = [
        r for r in relationships 
        if r not in relationships_to_remove
    ]
    
    return cleaned_relationships, relationships_to_remove

✅ 3.3 完整性验证

def validate_referential_integrity(relationship, connection):
    """
    验证外键的参照完整性
    
    检查是否存在孤儿记录 (orphan records)
    """
    fk_table = relationship['fk_table']
    fk_column = relationship['fk_column']
    pk_table = relationship['pk_table']
    pk_column = relationship['pk_column']
    
    # 查找孤儿记录
    query = f"""
        SELECT COUNT(*) as orphan_count,
               (SELECT COUNT(*) FROM {fk_table}) as total_count
        FROM {fk_table} fk
        WHERE NOT EXISTS (
            SELECT 1 FROM {pk_table} pk
            WHERE pk.{pk_column} = fk.{fk_column}
        )
    """
    
    result = execute_query(query, connection)[0]
    orphan_count = result[0]
    total_count = result[1]
    
    orphan_ratio = orphan_count / total_count if total_count > 0 else 0
    
    relationship['orphan_count'] = orphan_count
    relationship['orphan_ratio'] = orphan_ratio
    
    # 分级判断
    if orphan_ratio == 0:
        relationship['integrity_status'] = 'perfect'
        relationship['integrity_message'] = '无孤儿记录'
    elif orphan_ratio < 0.01:  # <1%
        relationship['integrity_status'] = 'good'
        relationship['integrity_message'] = \
            f'{orphan_count}条孤儿记录 ({orphan_ratio:.2%})'
    elif orphan_ratio < 0.05:  # 1-5%
        relationship['integrity_status'] = 'acceptable'
        relationship['integrity_message'] = \
            f'{orphan_count}条孤儿记录,需要数据清洗'
    else:  # >5%
        relationship['integrity_status'] = 'poor'
        relationship['integrity_message'] = \
            f'{orphan_count}条孤儿记录,可能不是真正的外键'
        relationship['final_score'] *= 0.7  # 降低置信度
    
    return relationship

def generate_final_output(relationships, primary_keys):
    """
    生成最终的JSON输出
    """
    output = {
        'metadata': {
            'timestamp': datetime.now().isoformat(),
            'total_tables': len(set(
                [r['fk_table'] for r in relationships] + 
                [r['pk_table'] for r in relationships]
            )),
            'algorithm_version': 'HoPF-v1.0',
        },
        'primary_keys': [],
        'foreign_keys': [],
        'warnings': [],
        'statistics': {}
    }
    
    # 主键输出
    for pk in primary_keys:
        output['primary_keys'].append({
            'table': pk['table'],
            'columns': pk['columns'],
            'confidence': pk['final_score'] / 10,
            'uniqueness': pk['uniqueness_ratio'],
            'reason': f"唯一性{pk['uniqueness_ratio']:.2%}, " + 
                     f"列数{len(pk['columns'])}"
        })
    
    # 外键输出
    for fk in relationships:
        if fk['status'] == 'accepted':
            output['foreign_keys'].append({
                'fk_table': fk['fk_table'],
                'fk_columns': fk['fk_column'].split(','),
                'pk_table': fk['pk_table'],
                'pk_columns': fk['pk_column'].split(','),
                'confidence': fk['final_score'] / 10,
                'relationship_type': infer_relationship_type(fk),
                'orphan_ratio': fk.get('orphan_ratio', 0),
                'integrity_status': fk.get('integrity_status', 'unknown'),
                'score_breakdown': fk['score_breakdown']
            })
        else:
            # 警告信息
            output['warnings'].append({
                'type': fk['status'],
                'fk': f"{fk['fk_table']}.{fk['fk_column']}",
                'pk': f"{fk['pk_table']}.{fk['pk_column']}",
                'score': fk['final_score'],
                'reason': fk.get('conflict_reason') or \
                         fk.get('cycle_warning') or \
                         fk.get('integrity_message')
            })
    
    # 统计信息
    output['statistics'] = {
        'total_primary_keys': len(output['primary_keys']),
        'total_foreign_keys': len(output['foreign_keys']),
        'high_confidence_fks': len([
            fk for fk in output['foreign_keys'] 
            if fk['confidence'] >= 0.8
        ]),
        'warnings_count': len(output['warnings']),
        'avg_confidence': sum(
            fk['confidence'] for fk in output['foreign_keys']
        ) / len(output['foreign_keys']) if output['foreign_keys'] else 0
    }
    
    return output


def infer_relationship_type(fk):
    """推断关系类型 (1:1, 1:N, N:M)"""
    fk_distinct = fk['stats']['fk_distinct']
    fk_total = fk['stats']['fk_total']
    pk_distinct = fk['stats']['pk_distinct']
    
    fk_uniqueness = fk_distinct / fk_total
    
    if fk_uniqueness >= 0.99:
        return '1:1'  # 一对一
    elif fk_distinct / pk_distinct < 0.3:
        return 'N:M'  # 多对多(可能需要中间表)
    else:
        return 'N:1'  # 多对一(最常见)

📊 算法评估

✅ 优势

  • 自动化程度高: 无需人工标注,自动发现关系
  • 准确率高: 多维度评分,准确率85-95%
  • 支持复合键: 能发现多列组合的主外键
  • 可解释性强: 提供详细的评分依据
  • 容错能力: 处理数据质量问题,如孤儿记录
  • 性能优化: 剪枝、采样、并行化策略

❌ 局限性

  • 数据质量依赖: 脏数据会影响判断准确性
  • 语义理解有限: 无法理解复杂业务规则
  • 计算开销: 大规模数据处理耗时较长
  • 隐式关系遗漏: 基于时间窗口的Join无法识别
  • 命名规范依赖: 不规范命名降低准确性
  • 多态关联: 一个外键指向多表的情况难处理
