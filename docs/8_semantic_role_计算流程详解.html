关键步骤说明

  • Step 1: CLI 命令解析，加载 configs/metadata_config.yaml
  • Step 2: 初始化 MetadataGenerator，创建 MetadataProfiler 实例
  • Step 3: 从 DDL 文件或数据库提取表结构元数据
  • Step 4: 执行数据采样（默认 1000 行），计算统计信息（唯一性、空值率、长度等）
  • Step 5: 调用 profiler._profile_columns() 为每个列生成画像
  • Step 6: 在 _classify_semantics() 中按优先级顺序判定 semantic_role
  • Step 7: 输出 JSON 文件到 output/json/ 目录

2. Semantic Role 判定顺序与优先级

配置参数

配置项 YAML 路径 默认值

为何最高优先级？ 复杂类型是基于数据类型的硬约束判定，不依赖命名模式。如果放在 audit/datetime 之后，audit_log (jsonb) 会因命名模式被误判为 audit，created_data (json) 会被误判为 datetime。类型硬约束必须优先于命名模式。
重要用途： Complex 类型字段不适合作为外键或逻辑主键，会在关系发现和逻辑主键检测中被过滤。

示例

✅ audit_log (jsonb) → complex (新顺序修复：不再被 audit 命名模式误判) ✅ metadata (jsonb) → complex ✅ tags (array) → complex ✅ config (json) → complex ✅ content (bytea) → complex ✅ location (point) → complex ✅ items (integer[]) → complex (数组后缀) ❌ description (text) → 继续后续判定

✅ 已实现： 规则2（high_uniqueness）新增"指标词拦截"机制。当唯一性>0.95的列命中指标词（amount/price/cost/count等）时，不会在规则2直接判定为identifier，而是继续走规则3进行命名特征检查，有效避免 amount (integer, 唯一性>0.95) 等指标字段被误判。详见 7_identifier_high_uniqueness_指标词拦截_简版方案.md

判定流程（多层过滤）

  • 规则0a：命名排除
    • 检查 identifier_exclude_keywords（如 name, desc, remark）
    • 命中则直接返回 False（不是 identifier）
  • 规则0b：类型白名单
    • 调用 _is_identifier_type() 检查数据类型
    • 不在白名单中直接返回 False
    • 白名单：integer, bigint, varchar, uuid, numeric/decimal (scale=0)
  • 规则1：物理约束（最高优先级）
    • 检查 PK / FK / UNIQUE constraint
    • 命中则返回 True，置信度 1.0
  • 规则2：高唯一性（逻辑推断）+ 指标词拦截
    • 条件：uniqueness > high_uniqueness_threshold AND non_null_rate > min_non_null_rate
    • 默认阈值：high_uniqueness_threshold=0.95, min_non_null_rate=0.80（可配置）
    • 指标词拦截（新增）：若列名包含指标关键词（amount/price/cost/revenue/sales/profit/count/qty/quantity/number/rate/ratio/percent/percentage），则不在此规则直接判定，继续走规则3
    • 未命中指标词时返回 True，置信度 0.95
  • 规则3：命名特征（逻辑推断）
    • 检查关键词：id, uid, code, key, pk, fk, no, num, number, sn, serial, seq, uuid, guid, ref, reference, identifier
    • 如有统计数据且 uniqueness > low_uniqueness_threshold（默认0.05，可配置）：置信度 0.85
    • 如有统计数据但 uniqueness ≤ low_uniqueness_threshold：置信度 0.75（仍识别为identifier）
    • 无统计数据：置信度 0.75

# profiler.py: 第973-1064行（_is_identifier + _has_* 系列方法） def _is_identifier( self, column: ColumnInfo, stats: Optional[Dict], struct_flags: StructureFlags, ) -> Tuple[bool, float, Optional[str], List[str]]: """综合判断是否为 identifier 按优先级依次检查： 0. 命名排除规则（name/desc 等描述性字段） 1. 数据类型白名单（前置过滤） 2. 物理约束（PK/FK/UNIQUE） 3. 统计特征（唯一性>high_uniqueness_threshold + 非空率>min_non_null_rate） 4. 命名特征（关键词 + 唯一性>low_uniqueness_threshold） 返回: (是否identifier, 置信度, 匹配的模式/关键词, 推断依据列表) """ # 规则0a: 命名排除 if any(kw in column_name for kw in self.config.identifier_exclude_keywords): return (False, 0.0, None, []) # 规则0b: 类型白名单 if not self._is_identifier_type(column): return (False, 0.0, None, []) # 规则1: 物理约束 has_constraint, conf, reason = self._has_physical_constraint(struct_flags) if has_constraint: return (True, conf, reason, [reason]) # 规则2: 高唯一性 has_high_uniq, uniq_conf, uniq_reason = self._has_high_uniqueness(stats) if has_high_uniq: # 新增：指标词拦截（子串包含匹配） metric_keywords = ["amount", "price", "cost", "revenue", "sales", "profit", "count", "qty", "quantity", "number", "rate", "ratio", "percent", "percentage"] if any(kw in lower_name for kw in metric_keywords): # 包含指标词，不在此处判定，继续走规则3 pass else: # 未包含指标词，直接返回 identifier return (True, uniq_conf, "logical_primary_key", [uniq_reason]) # 规则3: 命名特征 has_naming, naming_conf, matched_kw, reason = self._has_identifier_naming(column_name, stats) if has_naming: return (True, naming_conf, matched_kw, [reason]) return (False, 0.0, None, [])
