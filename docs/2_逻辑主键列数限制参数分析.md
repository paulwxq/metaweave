# 逻辑主键列数限制参数分析

## 问题

在执行以下命令时：
```bash
metaweave metadata --config configs/metadata_config.yaml --step json
metaweave metadata --config configs/metadata_config.yaml --step json_llm
```

这两个命令在生成逻辑主键时，是否有参数控制最多支持的逻辑主键列数？

## 答案

**是的，存在一个参数控制逻辑主键的最大列数。**

---

## 1. 配置参数位置

### 1.1 配置文件

**文件路径：** `configs/metadata_config.yaml`

**配置节：** `logical_key_detection`

```yaml
# 逻辑主键识别配置
logical_key_detection:
  enabled: true
  min_confidence: 0.8
  max_combinations: 3  # 最多考虑 3 字段组合
  
  # 主键字段名模式
  name_patterns:
    - id
    - code
    - key
    - no
    - number
    - pk
```

**关键参数：** `max_combinations: 3`

### 1.2 参数说明

| 参数名称 | 当前值 | 默认值 | 说明 |
|---------|-------|-------|------|
| `max_combinations` | 3 | 3 | **最多考虑的字段组合数**<br>用于控制复合逻辑主键的最大列数<br>例如：max_combinations=3 表示最多考虑 3 个字段的组合 |
| `min_confidence` | 0.8 | 0.7 | **最小置信度阈值**<br>只有置信度 ≥ 此阈值的候选才会被保留 |

```267:311:metaweave/core/metadata/logical_key_detector.py
    def _analyze_composite_keys(
        self,
        metadata: TableMetadata,
        sample_data: pd.DataFrame
    ) -> List[LogicalKey]:
        """分析复合键候选
        
        Args:
            metadata: 表元数据
            sample_data: 样本数据（用于计算组合的唯一性）
            
        Returns:
            复合键逻辑主键候选列表
        """
        candidates = []
        
        # 获取适合作为复合键组成的字段（只排除 metric）
        suitable_columns = self._get_suitable_columns_for_composite_key(metadata)
        
        if len(suitable_columns) < 2:
            logger.info("适合作为复合键组成的字段少于2个，跳过复合键分析")
            return candidates
        
        # 只对适合的字段生成组合
        for size in range(2, min(self.max_combinations + 1, len(suitable_columns) + 1)):
            for combo in combinations(suitable_columns, size):
                combo_list = list(combo)
                
                # 使用 sample_data 计算组合的唯一性和空值率
                # 注意：组合的统计信息无法从单个字段的统计中推导，必须重新计算
                uniqueness = calculate_uniqueness(sample_data, combo_list)
                null_rate = calculate_null_rate(sample_data, combo_list)
                
                # 复合键候选的条件：唯一度 = 100% 且无空值
                if uniqueness == 1.0 and null_rate == 0.0:
                    candidate = LogicalKey(
                        columns=combo_list,
                        uniqueness=uniqueness,
                        null_rate=null_rate,
                        confidence_score=0.0  # 后续计算
                    )
                    candidates.append(candidate)
                    logger.debug(f"复合键候选: {combo_list} (唯一度={uniqueness})")
        
        return candidates
```

## 3. 参数效果分析

### 3.1 当前配置效果

**配置值：** `max_combinations: 3`

**实际效果：**
- **单列逻辑主键：** 始终分析（不受此参数影响）
- **2 列复合键：** ✅ 会分析
- **3 列复合键：** ✅ 会分析
- **4 列及以上：** ❌ 不会分析

### 3.2 代码逻辑

```python
for size in range(2, min(self.max_combinations + 1, len(suitable_columns) + 1)):
```

**解释：**
- `range(2, min(self.max_combinations + 1, len(suitable_columns) + 1))`
- 当 `max_combinations = 3` 时：
  - `range(2, min(4, len(suitable_columns) + 1))`
  - 如果适合的字段数 ≥ 3，则 `range(2, 4)`，即 `size ∈ [2, 3]`
  - 如果适合的字段数 = 2，则 `range(2, 3)`，即 `size = 2`

### 3.3 修改示例

**场景 1：只考虑 2 列复合键**
```yaml
logical_key_detection:
  max_combinations: 2  # 只考虑 2 列组合
```

**场景 2：考虑最多 4 列复合键**
```yaml
logical_key_detection:
  max_combinations: 4  # 最多考虑 4 列组合
```

**⚠️ 注意事项：**
- 列数越多，组合数呈指数增长，计算时间会显著增加
- 建议保持在 2-3 列，超过 3 列的复合键在实际数据建模中较少见

## 4. 总结

### 4.1 问题答案

**问题：** 在生成逻辑主键时，是否有参数控制最多支持的逻辑主键列数？

**答案：**
- ✅ **是的，有参数控制**
- **参数名：** `max_combinations`
- **配置位置：** `configs/metadata_config.yaml` 的 `logical_key_detection` 节
- **当前值：** 3
- **默认值：** 3（代码中的 fallback）
- **参数类型：** 配置文件参数（非硬编码）

### 4.2 参数适用范围

| 命令 | 是否使用此参数 |
|-----|--------------|
| `metaweave metadata --step json` | ✅ 使用 |
| `metaweave metadata --step json_llm` | ✅ 使用 |
| `metaweave metadata --step rel` | ❌ 不使用（关系发现阶段） |
| `metaweave metadata --step rel_llm` | ❌ 不使用（关系发现阶段） |

**说明：**
- `json` 和 `json_llm` 步骤都会执行逻辑主键检测（`LogicalKeyDetector`）
- 这两个步骤使用相同的配置参数 `logical_key_detection.max_combinations`
- 关系发现步骤（`rel`/`rel_llm`）会**读取**已生成的逻辑主键，但不会重新生成

### 4.3 关系发现阶段的相关参数

**配置节：** `composite`

```yaml
# 复合键候选配置
composite:
  max_columns: 3  # 最多考虑3列的复合键
  logical_key_min_confidence: 0.8  # 逻辑主键最低置信度
```

**说明：**
- `composite.max_columns`: 关系发现阶段使用的复合键最大列数（与 `logical_key_detection.max_combinations` 含义类似，但用于不同阶段）
- `composite.logical_key_min_confidence`: 关系发现阶段收集逻辑主键时的最低置信度阈值（过滤低质量逻辑主键）

**⚠️ 重要：**
- **生成阶段**（`json`/`json_llm`）：使用 `logical_key_detection.max_combinations`
- **使用阶段**（`rel`/`rel_llm`）：使用 `composite.max_columns` 和 `composite.logical_key_min_confidence`
- 建议保持两个参数一致，避免生成了 3 列逻辑主键但关系发现阶段只使用 2 列的情况

## 5. 配置建议

### 5.1 当前配置（推荐）

```yaml
# 逻辑主键识别配置
logical_key_detection:
  max_combinations: 3  # 生成阶段：最多 3 列

# 复合键候选配置
composite:
  max_columns: 3  # 使用阶段：最多 3 列
```

**理由：**
- 3 列复合键在实际业务中较为常见（如：年份+地区+产品）
- 计算复杂度可控（组合数 = C(n, 2) + C(n, 3)）
- 两个阶段参数保持一致，避免信息丢失

### 5.2 性能优化配置

如果表字段数很多，且性能受限，可以减少列数：

```yaml
logical_key_detection:
  max_combinations: 2  # 只考虑 2 列复合键

composite:
  max_columns: 2
```

### 5.3 高质量分析配置

如果需要更全面的分析，且性能充足：

```yaml
logical_key_detection:
  max_combinations: 4  # 最多 4 列

composite:
  max_columns: 4
```

**⚠️ 警告：** 4 列及以上的复合键会导致计算时间显著增加，仅在必要时使用。
