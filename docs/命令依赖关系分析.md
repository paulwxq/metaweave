# MetaWeave 命令依赖关系分析

本文档详细分析 `metaweave metadata` 各个步骤命令的依赖关系、数据来源和资源访问情况。

## 📊 命令依赖关系概览表

| 命令步骤 | 数据库访问 | LLM访问 | 依赖文件输入 | 输出目录 | 主要功能 |
|---------|-----------|---------|------------|---------|---------|
| **ddl** | ✅ 是<br/>- 查询表结构<br/>- 查询约束索引<br/>- 采样数据（可选） | ❌ 否 | 无 | `output/ddl/` | 从数据库提取表结构生成DDL文件 |
| **json** | ✅ 是<br/>- 查询行数(COUNT)<br/>- 采样数据（可选） | ❌ 否 | `output/ddl/*.sql` | `output/json/` | 基于DDL+数据库采样生成表/列数据画像 |
| **json_llm** | ✅ 是<br/>- 提取表元数据<br/>- 采样数据 | ✅ 是<br/>- 推断分类<br/>- 推断业务主题<br/>- 生成注释 | `output/ddl/*.sql`<br/>（仅检查存在性） | `output/json_llm/` | 基于数据库元数据+采样，LLM增强生成简化版JSON |
| **rel** | ✅ 是<br/>- 采样数据评分<br/>- 计算包含率<br/>- 计算Jaccard指数 | ❌ 否 | `output/json/*.json` | `output/rel/` | 读取json，通过算法+数据库采样发现表关系 |
| **rel_llm** | ✅ 是<br/>- 采样数据评分<br/>- 验证关系 | ✅ 是<br/>- LLM推断关系 | `output/json_llm/*.json` | `output/rel/` | 读取json_llm，LLM推断+数据库采样发现关系 |
| **cql** | ❌ 否 | ❌ 否 | `output/json/*.json`<br/>`output/rel/*.json` | `output/cql/` | 纯文件读取，生成Neo4j CQL导入脚本 |
| **cql_llm** | ❌ 否 | ❌ 否 | `output/json_llm/*.json`<br/>`output/rel/*.json` | `output/cql/` | 纯文件读取，基于LLM增强数据生成CQL |
| **md** | ✅ 是<br/>- 查询表结构<br/>- 采样数据 | ❌ 否<br/>（可选启用） | 无 | `output/md/` | 直接从数据库生成Markdown文档 |

## 🔍 详细依赖说明

### 1️⃣ ddl 步骤

**命令**: `metaweave metadata --config configs/metadata_config.yaml --step ddl`

**数据来源**:
- ✅ **必须连接数据库**
  - 查询 `information_schema` 获取表结构
  - 查询约束和索引信息
  - 采样数据生成样例记录（如果配置启用）

**LLM使用**: ❌ 不需要（可选启用LLM生成注释）

**依赖文件**: 无

**输出**: `output/ddl/{schema}.{table}.sql`
- DDL 语句
- 约束和索引定义
- COMMENT 语句
- `/* SAMPLE_RECORDS {...} */` 注释块（包含采样数据）

**代码位置**: `metaweave/core/metadata/generator.py:_process_table_from_db()`

---

### 2️⃣ json 步骤

**命令**: `metaweave metadata --config configs/metadata_config.yaml --step json`

**数据来源**:
- ✅ **访问数据库**（代码确认）
  - 第399-409行：执行 `SELECT COUNT(*)` 查询真实行数
  - 第411-414行：如果启用采样，调用 `connector.sample_data()` 从数据库采样
- 📄 读取 DDL 文件解析表结构

**LLM使用**: ❌ 不需要

**依赖文件**: `output/ddl/*.sql`

**输出**: `output/json/{schema}.{table}.json`
- 表画像 (`table_profile`)
- 列画像 (`column_profiles`)
- 逻辑主键 (`candidate_logical_primary_keys`)
- 统计信息（基于采样数据）

**代码位置**: `metaweave/core/metadata/generator.py:_process_table_from_ddl()`

**关键代码**:
```python
# 第399-409行：查询行数
count_sql = f"SELECT COUNT(*) as row_count FROM {schema}.{table}"
query_result = self.connector.execute_query(count_sql, fetch_one=True)

# 第411-414行：采样数据
if self.sampling_enabled:
    sample_data = self.connector.sample_data(schema, table, self.sample_size)
    self._apply_column_statistics(metadata, sample_data)
```

---

### 3️⃣ json_llm 步骤

**命令**: `metaweave metadata --config configs/metadata_config.yaml --step json_llm`

**数据来源**:
- ✅ **必须连接数据库**（代码确认）
  - 第276行：`self.extractor.extract_all(schema, table)` - 从数据库提取元数据
  - 第280行：`self.connector.sample_data(schema, table, self.sample_size)` - 从数据库采样
- 📄 DDL 文件仅用于检查目录存在性

**LLM使用**: ✅ **必须**
- 推断 `table_category`（表分类）
- 推断 `table_domains`（业务主题）
- 生成表和列的中文注释

**依赖文件**: `output/ddl/*.sql`（仅检查目录存在）

**输出**: `output/json_llm/{schema}.{table}.json`

**代码位置**: `metaweave/core/metadata/llm_json_generator.py`
- `generate_all_from_ddl()` (125-137行)
- `_generate_single_table()` (273-283行)

**关键代码**:
```python
# 第276-280行：访问数据库
metadata: TableMetadata = self.extractor.extract_all(schema, table)  # 数据库提取
sample_df: pd.DataFrame = self.connector.sample_data(schema, table, self.sample_size)  # 数据库采样
```

---

### 4️⃣ rel 步骤

**命令**: `metaweave metadata --config configs/metadata_config.yaml --step rel`

**数据来源**:
- 📄 读取 `output/json/*.json` 文件
- ✅ **必须连接数据库** - 评分阶段采样
  - 采样源表和目标表数据
  - 计算 `inclusion_rate`（包含率）
  - 计算 `jaccard_index`（Jaccard相似度）
  - 计算唯一性和基数

**LLM使用**: ❌ 不需要

**依赖文件**: `output/json/*.json`

**输出**: `output/rel/relationships_global.json`

**代码位置**: 
- `metaweave/core/relationships/pipeline.py:discover()`
- `metaweave/core/relationships/scorer.py:_sample_and_calculate_inclusion()`

**关键代码**:
```python
# scorer.py 第270-285行：数据库采样
source_sql = f'''
    SELECT {source_col_expr}
    FROM "{source_schema}"."{source_table}"
    LIMIT %s
'''
source_rows = self.connector.execute_query(source_sql, (self.sample_size,))

target_sql = f'''
    SELECT {target_col_expr}
    FROM "{target_schema}"."{target_table}"
    LIMIT %s
'''
target_rows = self.connector.execute_query(target_sql, (self.sample_size,))
```

---

### 5️⃣ rel_llm 步骤

**命令**: `metaweave metadata --config configs/metadata_config.yaml --step rel_llm`

**数据来源**:
- 📄 读取 `output/json_llm/*.json` 文件
- ✅ **必须连接数据库** - 评分阶段采样（与rel步骤相同）
- 🤖 **必须LLM** - 推断表之间的关联关系

**LLM使用**: ✅ **必须**
- 分析两个表的JSON数据和样例
- 推断可能的关联关系（单列或复合列）

**依赖文件**: `output/json_llm/*.json`

**输出**: `output/rel/relationships_global.json`

**代码位置**: `metaweave/core/relationships/llm_relationship_discovery.py`

---

### 6️⃣ cql 步骤

**命令**: `metaweave metadata --config configs/metadata_config.yaml --step cql`

**数据来源**:
- 📄 读取 `output/json/*.json`（表和列信息）
- 📄 读取 `output/rel/relationships_global.json`（关系信息）

**数据库访问**: ❌ **不访问** - 纯文件读取

**LLM使用**: ❌ 不需要

**依赖文件**: 
- `output/json/*.json`
- `output/rel/relationships_global.json`

**输出**: `output/cql/*.cypher`
- `01_nodes_tables.cypher`
- `02_nodes_columns.cypher`
- `04_rels_has_column.cypher`
- `05_rels_join_on.cypher`
- `00_import_all.cypher`（汇总脚本）

**代码位置**: `metaweave/core/cql_generator/generator.py`

---

### 7️⃣ cql_llm 步骤

**命令**: `metaweave metadata --config configs/metadata_config.yaml --step cql_llm`

**数据来源**:
- 📄 读取 `output/json_llm/*.json`（LLM增强的表和列信息）
- 📄 读取 `output/rel/relationships_global.json`（关系信息）

**数据库访问**: ❌ **不访问** - 纯文件读取

**LLM使用**: ❌ 不需要（读取阶段不需要，数据已在json_llm中）

**依赖文件**: 
- `output/json_llm/*.json`
- `output/rel/relationships_global.json`

**输出**: `output/cql/*.cypher`（与cql步骤相同）

**代码位置**: `metaweave/core/cql_generator/generator.py`

---

### 8️⃣ md 步骤

**命令**: `metaweave metadata --config configs/metadata_config.yaml --step md`

**数据来源**:
- ✅ **必须连接数据库**
  - 直接查询表结构
  - 采样数据用于生成示例

**LLM使用**: ❌ 不需要（可选启用LLM生成注释）

**依赖文件**: 无（独立执行）

**输出**: `output/md/{schema}.{table}.md`

**代码位置**: `metaweave/core/metadata/generator.py:_process_table_from_db()`

## 🔄 执行流程图

### 非LLM流程（标准流程）

```
┌─────────────────┐
│  PostgreSQL DB  │ (数据源)
└────────┬────────┘
         │
         ▼ 查询表结构+采样
    ┌────────┐
    │  ddl   │ Step 1: 生成DDL文件
    └───┬────┘
        │ 输出: output/ddl/*.sql
        ▼ 读取DDL
    ┌────────┐
    │  json  │ Step 2: 生成数据画像
    └───┬────┘ (还需访问DB: COUNT查询+采样)
        │ 输出: output/json/*.json
        ▼ 读取JSON
    ┌────────┐
    │  rel   │ Step 3: 发现表关系
    └───┬────┘ (还需访问DB: 采样评分)
        │ 输出: output/rel/*.json
        ▼ 读取JSON+REL
    ┌────────┐
    │  cql   │ Step 4: 生成Neo4j CQL
    └────────┘ (纯文件操作，不访问DB)
         输出: output/cql/*.cypher

    ┌────────┐
    │   md   │ Step 5: 生成Markdown
    └────────┘ (直接访问DB，独立流程)
         输出: output/md/*.md
```

### LLM增强流程

```
┌─────────────────┐        ┌──────────┐
│  PostgreSQL DB  │        │ LLM API  │
└────────┬────────┘        └─────┬────┘
         │                       │
         ▼ 查询表结构+采样        │
    ┌────────┐                  │
    │  ddl   │ Step 1           │
    └───┬────┘                  │
        │ 输出: ddl/*.sql       │
        ▼ 检查DDL存在           │
    ┌────────┐                  │
    │json_llm│ Step 2           │
    └───┬────┘ ◄─────────────────┘推断分类/主题/注释
        │ (访问DB: 提取元数据+采样)
        │ 输出: json_llm/*.json
        ▼ 读取JSON_LLM
    ┌────────┐
    │rel_llm │ Step 3
    └───┬────┘ ◄─────────────────┐推断关系
        │ (访问DB: 采样评分)    │
        │ 输出: rel/*.json      │
        ▼ 读取JSON_LLM+REL     │
    ┌────────┐                  │
    │cql_llm │ Step 4           │
    └────────┘ (纯文件操作)     │
         输出: cql/*.cypher      │
```

## 📋 数据库访问总结

| 步骤 | 是否访问数据库 | 访问目的 |
|-----|--------------|---------|
| ddl | ✅ 是 | 查询表结构、约束、索引、采样数据 |
| json | ✅ 是 | COUNT查询行数、采样数据计算统计 |
| json_llm | ✅ 是 | 提取元数据、采样数据（不读取DDL内容） |
| rel | ✅ 是 | 采样数据用于关系评分 |
| rel_llm | ✅ 是 | 采样数据用于关系评分 |
| cql | ❌ 否 | 纯文件读取 |
| cql_llm | ❌ 否 | 纯文件读取 |
| md | ✅ 是 | 查询表结构、采样数据 |

**⚠️ 重要发现**: 除了 `cql` 和 `cql_llm` 两个步骤，**其他所有步骤都需要访问数据库**！

## 🤖 LLM访问总结

| 步骤 | 是否访问LLM | 使用目的 |
|-----|-----------|---------|
| ddl | ❌ 否 | 不需要（可选启用） |
| json | ❌ 否 | 不需要 |
| json_llm | ✅ 是 | 推断表分类、业务主题、生成注释 |
| rel | ❌ 否 | 不需要 |
| rel_llm | ✅ 是 | 推断表之间的关联关系 |
| cql | ❌ 否 | 不需要 |
| cql_llm | ❌ 否 | 不需要（数据已在json_llm中） |
| md | ❌ 否 | 不需要（可选启用） |

## 🎯 执行建议

### 标准流程（不使用LLM）
```bash
# Step 1: 生成DDL（需要DB）
metaweave metadata --config configs/metadata_config.yaml --step ddl

# Step 2: 生成JSON画像（需要DB）
metaweave metadata --config configs/metadata_config.yaml --step json

# Step 3: 发现关系（需要DB）
metaweave metadata --config configs/metadata_config.yaml --step rel

# Step 4: 生成CQL（不需要DB）
metaweave metadata --config configs/metadata_config.yaml --step cql

# Step 5: 生成MD（需要DB，独立执行）
metaweave metadata --config configs/metadata_config.yaml --step md
```

### LLM增强流程（需要LLM API Key）
```bash
# Step 1: 生成DDL（需要DB）
metaweave metadata --config configs/metadata_config.yaml --step ddl

# Step 2: LLM增强JSON（需要DB + LLM）
metaweave metadata --config configs/metadata_config.yaml --step json_llm

# Step 3: LLM发现关系（需要DB + LLM）
metaweave metadata --config configs/metadata_config.yaml --step rel_llm

# Step 4: 生成CQL（不需要DB）
metaweave metadata --config configs/metadata_config.yaml --step cql_llm
```

### 一次性执行
```bash
# 执行所有标准步骤
metaweave metadata --config configs/metadata_config.yaml --step all
```

## ⚠️ 注意事项

1. **数据库连接**: 
   - 除了 `cql` 和 `cql_llm`，所有步骤都需要配置正确的数据库连接
   - 即使读取DDL文件，仍需要数据库采样来计算统计指标

2. **LLM配置**:
   - `json_llm` 和 `rel_llm` 步骤需要配置 LLM API Key
   - LLM流程比标准流程慢，但语义理解更准确

3. **依赖顺序**:
   - `json` 依赖 `ddl`
   - `json_llm` 依赖 `ddl`（目录存在性）
   - `rel` 依赖 `json`
   - `rel_llm` 依赖 `json_llm`
   - `cql` 依赖 `json` + `rel`
   - `cql_llm` 依赖 `json_llm` + `rel`
   - `md` 独立执行

4. **性能考虑**:
   - 标准流程更快，适合大规模数据
   - LLM流程更智能，适合需要语义理解的场景
   - 可以根据需求混合使用两种流程
