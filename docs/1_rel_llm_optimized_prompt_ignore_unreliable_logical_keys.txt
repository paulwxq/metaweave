## 任务（如何判断是否有关联）
请综合以下证据判断并给出最可能的关联字段（可 0 个、1 个或多个）：
1) 字段名相同或相似（例如 *_id、*_code、*_no 等；以及语义相近的命名）

## 输出格式（必须严格遵守）
你必须返回一个 JSON 对象，且仅包含一个顶层字段：relationships。
relationships 是一个数组。数组里每个元素是一个关系对象，关系对象只能是以下两种结构之一：

### A. 单列关联（type 为 single_column）

### 方向约定（请遵守）
如果你认为存在“被引用表(字典/维表/主数据)”与“引用表(事实/明细/记录)”：
- from_table = 被引用表（更像主表/字典/维表的一方，通常提供稳定编码/ID）
- to_table   = 引用表（更像事实/明细/事件的一方，包含外键字段）
字段方向同理：from_column(s) 为被引用列，to_column(s) 为引用列。

### 额外约束
1) 不要输出不存在的表名或列名；必须能在输入 JSON 中找到对应列。

3) 控制数量：最多输出 5 个最可能的关系（按可能性从高到低）。
